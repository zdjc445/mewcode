# Chapter 04 验收清单

## 安全边界

- [x] 已知递归强制删除、磁盘操作、系统电源操作、远程下载后执行、破坏性 Git 操作和 fork bomb 在执行前硬拒绝。
- [x] 文件工具路径和 `run_command.cwd` 被限制在启动工作目录内。
- [x] `..`、绝对路径越界、符号链接越界和 glob 越界被拒绝。
- [x] 硬拒绝和路径沙箱不能被规则、模式或计划批准覆盖。
- [x] Registry 和文件工具在 Scheduler 之后再次检查硬边界。

## 规则与模式

- [x] 用户和项目两层严格 YAML 配置已经接入 CLI。
- [x] 规则支持精确工具名及 `exact`、`glob`、`path_glob` 参数 matcher。
- [x] 会话规则优先于项目规则，项目规则优先于用户和永久规则。
- [x] 同层按 priority、`deny > ask > allow`、id 稳定决策。
- [x] `strict`、`default`、`permissive` 只决定未匹配调用。
- [x] 项目配置不能切换权限模式。

## HITL

- [x] 审批支持本次允许、本会话允许、永久允许和拒绝。
- [x] 本次允许不会授权下一调用。
- [x] 会话允许只存在于当前 `SecurityPolicyEngine` 生命周期。
- [x] 永久允许使用原子写入和 SHA-256 指纹。
- [x] 永久审批绑定项目根目录且不保存原始命令或文件正文。
- [x] 保存永久审批失败时不执行工具。

## 明确限制

- [x] 文档明确说明危险命令黑名单不是完整 Shell 解析器。
- [x] 文档明确说明当前没有操作系统级文件、网络或进程沙箱。
